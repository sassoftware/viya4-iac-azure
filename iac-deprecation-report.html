🐧 AKS Ubuntu OS and Node Image Support

This section summarizes Ubuntu OS SKU support, retirement dates, and node image constraints from current Azure AKS documentation.

Ubuntu OS SKU Support

OS SKU Supported Kubernetes Versions Default / Migration Behavior
Ubuntu All supported AKS Kubernetes versions Ubuntu 22.04 is the default for Kubernetes 1.25 to 1.34. Ubuntu 24.04 is the default for Kubernetes 1.35+.
Ubuntu2204 1.25 to 1.36 Versioned OS SKU for staying on or rolling back to Ubuntu 22.04.
Ubuntu2404 1.32 to 1.38 Versioned OS SKU for adopting Ubuntu 24.04 before Kubernetes 1.35 or for explicit pinning.

Ubuntu Retirement Milestones

Version Support Ends Image Removal Operational Impact
Ubuntu 20.04 2027-03-17 2027-03-17 No AKS support or security updates. Existing node images are deleted and node pools can no longer scale.
Ubuntu 22.04 2027-06-30 2028-04-30 After support ends, AKS stops producing new node images and security patches. After image removal, scaling and remediation operations fail.

Ubuntu Node Image Support

Variant Support Limitations
Ubuntu with containerd and Gen 1 Supported for Ubuntu node pools on VM sizes that only support Generation 1. Used only when the VM size does not support Gen 2.
Ubuntu with containerd and Gen 2 Default Ubuntu node image for VM sizes that support Generation 2. Selected by default when a VM supports both Gen 1 and Gen 2.
Ubuntu with containerd and FIPS Supported for FIPS-enabled Ubuntu node pools on Gen 1 and Gen 2 where supported by AKS. Not supported for Ubuntu 24.04+. Cannot be combined with Arm64 or CVM.
Ubuntu with containerd and Arm64 Supported for Arm64 Ubuntu node pools. Gen 2 only. Cannot be combined with FIPS, CVM, or Trusted Launch.
Ubuntu with containerd and CVM Supported for Confidential VM Ubuntu node pools on Ubuntu 20.04 and Ubuntu 24.04. Not supported for Ubuntu 22.04. Gen 2 only. Cannot be combined with FIPS, Arm64, or Trusted Launch.
Ubuntu with containerd and Trusted Launch Supported for Trusted Launch Ubuntu node pools. Gen 2 only. Cannot be combined with Arm64 or CVM.

Migration Notes

  • For OS SKU Ubuntu, AKS automatically moves the default Ubuntu version to 24.04 when the cluster upgrades to Kubernetes 1.35 or later.
  • Ubuntu 24.04 node images use containerd 2.0 by default, so workloads that depend on runtime behavior should be validated before migration.
  • Ubuntu 24.04 does not support FIPS in AKS at this time.

🖥️ AKS Azure Linux and Windows Support

This section summarizes Azure Linux and Windows OS SKU support, retirements, and node image constraints from current Azure AKS documentation.

Node Image Release Cadence

  • Linux node images are released weekly and Windows node images are released monthly.
  • New node images can take up to two weeks to roll out across all Azure regions.
  • Azure recommends automatic node image upgrades and planned maintenance to keep nodes current.

Azure Linux OS SKU Support

OS SKU Supported Kubernetes Versions Default / Migration Behavior
AzureLinux All supported AKS Kubernetes versions Azure Linux 2.0 is default for Kubernetes 1.27 to 1.31. Azure Linux 3.0 is default for Kubernetes 1.32+.
AzureLinux3 1.28 to 1.36 Versioned OS SKU for explicit migration to Azure Linux 3.0 without a Kubernetes upgrade.
AzureLinuxOSGuard 1.32 and above OS Guard images are upgraded through node image upgrades.
Flatcar All supported AKS Kubernetes versions Flatcar versions are delivered through node image upgrades.
AzureContainerLinux 1.34 and above Azure Container Linux versions are delivered through node image upgrades.

Windows OS SKU Support

OS SKU Supported Kubernetes Versions Default / Migration Behavior
Windows2019 1.14 to 1.32 Default Windows OS in Kubernetes 1.14 to 1.24.
Windows2022 1.23 to 1.34 Default Windows OS in Kubernetes 1.25 to 1.34.

Platform Retirement Milestones

Platform Support Ends Image Removal Operational Impact
Azure Linux 2.0 2025-11-30 2026-03-31 No new security updates after support ends, then node pools can no longer scale after image removal.
Windows Server 2019 2026-03-01 2027-04-01 Kubernetes 1.33+ cannot use Windows Server 2019. After image removal, scaling operations fail.
Windows Server 2022 2028-06-30 2029-06-30 Windows Server 2022 is not supported in Kubernetes 1.37+ and is fully removed later.
Windows Annual Channel (Preview) 2026-05-15 2027-05-15 No new node pools or security patches after support ends. Reimage and redeploy operations fail after removal.

Azure Linux and Windows Node Image Notes

Platform Variant Notes
Azure Linux Gen 1 / Gen 2 Standard node image for Azure Linux pools; Gen 2 is chosen when the VM supports both generations.
Azure Linux FIPS / Arm64 / FIPS+Arm64 / Trusted Launch / Pod Sandboxing Feature-specific variants exist, but combinations are constrained. Trusted Launch, Pod Sandboxing, FIPS, and Arm64 cannot all be combined freely.
Azure Linux OS Guard Gen 2 only OS Guard is preview-only and unavailable on Gen 1-only VM sizes.
Windows LTSC Gen 1 / Gen 2 Windows Server 2019 and 2022 use Gen 1 by default when a VM supports both Gen 1 and Gen 2. Windows Server 2025 selects Gen 2.
Windows Annual Channel Gen 1 / Gen 2 Preview channel only; use LTSC for long-term support.

Migration Notes

  • For Azure Linux, Azure recommends the default OS SKU AzureLinux so clusters move to the latest GA Azure Linux version with Kubernetes upgrades.
  • You can migrate Linux pools between supported OS SKUs with az aks nodepool update, but Windows OS SKUs are not supported by nodepool update and require adding node pools with the desired OS SKU.
  • Windows guidance in Azure docs recommends Windows2022 for new Windows node pools.
